const 对象必须初始化

const 对象仅在文件内生效, 原因是编译器会在编译时对变量进行替换，在编译时，要找到变量的初始值，因此，必须在每个用到变量的文件内存在对它的定义。

如果想在多个文件之间共享const对象，必须在变量定义之前添加extern关键字  如extern const int bufSize = fcn();

别处文件要用的话，也只能加extern关键字 extern const int bufSize;



引用的类型必须和被引用对象的类型保持一致

引用可能是const类型,但是其引用的对象不一定是const类型的

const 引用表示不允许通过引用改变其引用对象的值

顶层const 表示指针对象本身是一个常量，不可改变其值，而底层const表示指针指向的对象，是常量。

顶层const可以表示指针，对象等本身是个常量，而底层const表示指针等复合类型指向的基本类型是常量。

在进行拷贝动作时，可以不考虑顶层const,但必须考虑底层const

constexpr 常量表达式，编译器就能获取计算结果的表达式
